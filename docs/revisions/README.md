# Revisions

This folder tracks project changes after they are made.

## Purpose
- record what changed
- explain why it changed
- capture the impact of the change

## Structure
- `revision-template.md`: reusable change log entry
- revision entries named by date and change theme

## Naming
- `2026-04-20-hud-cache-cleanup.md`
- `2026-04-20-input-refactor.md`

## Required Sections
- date
- change
- why
- impact

## Rules
- revision files describe completed changes, not planned work
- keep them short and factual
- link to related tasks or playtests when useful
- do not duplicate full design docs here; summarize and point back to the canonical doc
