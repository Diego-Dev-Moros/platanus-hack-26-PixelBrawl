# Tasks

This folder tracks active and completed development work.

## Purpose
- keep milestone goals visible
- break milestone work into small tasks
- record status and short implementation notes

## Structure
- `milestone-XX-name.md`: one milestone tracker
- `task-template.md`: template for individual tasks
- `milestone-template.md`: template for milestone trackers

Keep the folder flat unless the number of task files becomes hard to scan.

## Status Vocabulary
- `todo`
- `in progress`
- `done`
- `blocked`

Use the same status words everywhere.

## Rules
- milestone files own grouped work and progress snapshots
- task files own one concrete unit of work
- design snapshots do not belong here unless they are attached to active work
- if a file becomes historical instead of active, move or summarize it in `docs/revisions/`

## Naming
- milestone: `milestone-01-core-loop.md`
- task: `task-input-cleanup.md`

## Required Sections

### Milestone
- title
- objective
- scope
- status
- task list
- notes

### Task
- title
- related milestone
- status
- goal
- constraints
- notes
- outcome
